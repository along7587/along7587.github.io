A 神奇石板

描述：
haomie现在手上有一块大小为N行N列神奇石板，石板上有许多英文字母，而且可以用手划动某一行（或列）使得这一行（或列）向左或向右平移1个
位置（处于边缘的字母，如位于行首的字母向左移则会移动到行尾），又或者使得某一层顺（或逆）时针旋转90°，与此同时，经过上述操作后，发生移动的字母则会大小写互换（如a->A，B->b）。
由于haomie太过想要炫耀这块神奇石板，但又怕它被人看到后被带走做研究，于是他想要让你自己按下面要求写一个程序来体验一下这块神奇石板到底是怎么样的。

输入格式
第一行给出一个数字N(0<=N<=100)表示石板的大小为N*N，接下来的N行有N个字母表示刻在石板上的字母。
然后给出一个数字M（1<=M<=1000）表示进行多少次操作。
有如下操作：
print_it! ：将目前神奇石板的样子输出出来。
clockwise_rotation x：顺时针旋转第x层。
anticlockwise_rotation x：逆时针旋转第x层。
left_shift x：左移第x行。
right_shift x：右移第x行。
upper_shift x：上移第x列。
down_shift x：下移第x列。
上述操作除了”print_it!”以外，都为一个操作接一个数字x(x不会出现在合法范围外）。
输出格式
当前操作为”print_it!”时将神奇石板输出。（至少存在一个”print_it!”操作）。

按照题目要求模拟就可以了。记得自己试试样例中没有的操作还有中间的那个数变不变的情况

B sen数字魔咒

描述
蒟蒻小周进入SCAU之后，听说ACM校队里面有很多得道高人，她希望可以见到高人们并得到真传。

但是高人们神出鬼没，行踪不定，这让小周相当苦恼。

此时仙女教母sen突然出现，他告诉小周，只要破解了sen数字魔咒，就可以见到高人们了。

sen数字魔咒是这样的，sen会给一个整数n(1<=n<=100000)，和n个整数xi（0<=xi<=9），

问能否在n个数里面取出若干个数字组成一个可以被60整除的数y（也就是说y至少要由一个数字组成，最多由n个数字组成）。

你能帮助小周解开魔咒，见到高人们吗？

可以注意到，被60整除的前提必须要有0，比如60,120,180.又注意到0除于任何数都为零。则只要有零即可。

C 负责端茶递水的jinli_

描述
ICPC的一支队伍一般由三名队员组成，而众所周知，飞机队只有两个写题手：koo_1999和Fczzz，剩下的jinli_只负责端茶递水，捶背按摩，有时候还要帮忙买饭。于是koo_和Fczzz就特别喜欢欺负jinli_。今天他们又给jinli_出了一个题目，题目是这样子的：**
有一个长度为N的字符串S和长度为M的字符串T。
可以对串S进行以下三种操作：
1.在其第一个字母前面插入任意一个字母

2.在其最后一个字母后面插入任意一个字母

3.删除其第一个字母

进行尽量少的操作使得S串变成T串，虽然这个问题很简单，但是对于jinli_来说太难了，如果做不出来就要被koo_1999和Fczzz嘲笑，你能帮帮他吗？
1<=N,M<=10000

很明显的求最长公共子串，动态规划问题，当然暴力也可以解决。

D sen在玩塞尔达传说

描述
sen这段时间沉迷塞尔达，总是在直播间直播ns游戏塞尔达传说：旷野之息。

小se是sen直播间的忠实观众，因为小se觉得看sen挨打真的很有趣。

小se通过时间不长的观察，发现sen战斗的时候有一个习惯，那就是勤俭持家。

因为旷野之息中的武器有耐久度的设定，耐久度降至0后就会损坏（直接消失），无法继续使用。所以sen每次都用最垃圾的武器打怪。

这会sen又进了游戏中的一个战斗试炼，小se作为早早通关旷野之息的高端玩家，早就知道了里面怪物的详细数据，包括怪物的血量、持有的武器和武器的耐久度和攻击。

小se这会通过直播画面也知道了sen背包里武器和盾牌的情况，小se决定在直播间当一回预言家，准确预告sen打完这些怪物后背包里的装备。

但是很不巧，小se这会被人叫去LOL了，没有时间计算，所以请你来写一个程序帮忙算算最终sen背包的装备。

以下是战斗规则

sen会按顺序与怪物一一决斗。

sen的战斗可以被视为回合制战斗，sen先攻一次，怪物攻一次，之后继续重复这个流程。

怪物攻击时sen如果有盾牌会使用耐久度最低的盾牌防御，如果这样的盾牌有多个，会选择最早获得的那个。如果背包没有盾牌了也没关系，sen会使用空翻躲避攻击。

怪物攻击时怪物的武器不会减少耐久度

sen每次攻击会使用背包里攻击最低的武器，如果攻击最低的武器有多个，那么会使用耐久度最低的那个，如果耐久度最低仍有多个，会选择最早获得到的武器。

sen使用武器攻击时，武器的耐久度会-1，对怪物造成武器攻击力的伤害，当怪物血量降低至0或以下时怪物死亡。

武器耐久度降低至0时损坏，损坏的武器会从背包中消失。

sen使用盾牌防御时，盾牌的耐久度会减少怪物拥有的武器的攻击力数值。

sen在击杀怪物后会将怪物的武器据为己有，如果此时背包满了，背包最小攻击力的武器小于该武器时，会用该武器替换；

如果背包最小攻击力的武器等于该武器但是耐久度低于或等于怪物的武器时，也会用该武器替换；

值得一提的是，怪物身上也可能会有盾牌，但是sen技术还可以，所以战斗不用考虑怪物的盾牌。

但是sen会捡怪物的盾牌，盾牌的替换规则按照盾牌的耐久度，如果背包中耐久度最低的低于或等于怪物的盾牌，用该盾牌替换怪物的盾牌。

最后还有就是如果sen的武器全损坏了，攻击不了只能die了，最终背包什么武器盾牌也没有了

输入格式
第一行给出用例数t(1<=t<=100)

第二行开始给出每组数据用例

对于每组数据用例

第1行给出，sen背包的武器数n和盾牌数m，怪物数k，背包武器容量l和背包盾牌容量p(1<=n<=10,0<=m<=10,1<=k<=100,n<=l<=20,m<=p<=20)

第2~n+1行每行按获得顺序给出sen已有的武器名si和攻击力ai和耐久度bi(1<=ai<=128,1<=bi<=100)

第n+2~n+m+1行每行按获得顺序给出sen已有的盾牌名ti和耐久度ci(1<=ci<=999)

第n+m+2~n+m+k+1行给出每个怪物的血量bi，持有的武器名xi、攻击力yi、武器耐久度zi、盾牌名qi和盾牌耐久度wi（如果怪物没有盾牌，那么盾牌耐久度会是0）

盾牌名和武器名的长度不会超过20

（1000<=bi<=5000,1<=yi<=128,1<=zi<=100,0<=wi<=999)
以上全部数据均为整数
输出格式
对于每组数据用例

第一行输出sen背包中剩余的武器数量n

第2~n+1行按获得顺序输出sen背包中的武器名

第n+2行输出sen背包中剩余的盾牌数量m

第n+3~n+m+2行按获得顺序输出sen背包中的盾牌名

利用一个优先队列分别存武器，盾牌。然后打怪的时候先pop出。然后A一下怪，判断属性是否爆掉，然后再push回去。相当于A一下收一下。

E 古典文学部

描述
千反田在收拾古典文学部的储物室时，发现了一盒陈年已久的火柴。

“这显然是不能用了呀”，她思索道。

突然，千反田想到了个点子并跑到了折木前面。

“我很好奇！”

看着双手捧着火柴盒的千反田，折木揉了揉额头知道她又想到了什么问题。

折木是个节能主义者，看到了你从活动室前经过想邀请你一起解答问题，你能帮帮他吗。

现在有a根火柴，要求恰好用完这a个火柴摆出数位为n的数字，并要求这个数字最大，或者判断没有数字能满足条件。

数字0需要6根火柴。

数字1需要2根火柴。

数字2需要5根火柴。

数字3需要5根火柴。

数字4需要4根火柴。

数字5需要5根火柴。

数字6需要6根火柴。

数字7需要3根火柴。

数字8需要7根火柴。

数字9需要6根火柴。

输入格式
第一行一个整数T(1 <= T <= 100)，表示接下来有T个测试输入。

接下来T行，每行两个整数a，n(1 <= a,n <= 200)，涵义如上表示。
输出格式
输出T行数字，分别表示依次表示T个测试的答案。

对于每个测试的答案，若存在满足条件的数字则将其输出，否则输出-1。

这道题要求数字最大，那么显然当不同的数字需要相同数量的火柴时，我们取最大的数字。那么题目意思便转换成：用现有的火柴组成数字1,4,5,7,8,9可以组成的最大数字。从最特殊的条件便是全部为1的情况。然后从全部为1的情况开始模拟就可以了。

G 小弟弟的算数题

描述
Fczzz和大只佬是好朋友，大只佬有一个可爱的弟弟（以下简称为小弟弟），作为计算机专业的学生，大只佬经常写程序自动生成算术题给小弟弟做。

最近小弟弟学到了一些很神奇的运算并且讲给了大只佬听：

2*5=10

25*4=100

125*8=1000等等

小弟弟说这些数字相乘以后后面全部都是0，很神奇，并让大只佬出一些这种类型的题目给他做。大只佬想了一想，决定提高一点点算术题的难度，他编写了一段程序并且生成了n个小数。

大只佬决定每天从这n个小数里面选出2个小数，让小弟弟进行相乘的运算，因为要让小弟弟体验到神奇，当这两个小数进行相乘以后，小数点以后的部分必须全部为0。

例如 (0.25) * (4.0)=1.00000000······(小数点后全为0） 这是神奇的运算 , 但 (0.25)*(0.4)=0.1······(小数点后不全为0)这是不神奇的运算。

因为大只佬身兼数职，他不想每天都重新生成小数，他想知道生成的n个小数可以给小弟弟出多少天的题（每天出一题），于是他找来了Fczzz帮忙计算一下。

为了简化问题，Fczzz决定把问题简单化为：

生成了n个小数，为 X1,X2,X3,······，Xn. 求能选出多少组【i,j】满足：

1、1≤i≤n,1≤j≤n;

2、i小于j 并且 Xi*Xj满足上面定义的神奇的运算;

这题我比较肝，先是字符串读入后扫一遍，记录需要进位需要的2和5的对数，然后对每个数提取2和5的因数个数，存入二维数组当中，用个二维前缀和来匹配是否进位。时间复杂度O(n*logn)。
